Conclusions:
- The hard link and original file shared inode 1310805. After deleting original.txt, hardlink.txt remained accessible because the file data still had a link count of 1.
- The symbolic link had a different inode (1310806) and depended on the pathname original.txt. When the target was removed, the symbolic link became broken.
- Hard links are alternate directory entries to the same inode, while symlinks store a path to the target file.
